# SGPD - Sistema de Gestão de Processos e Demandas 🚀

![Status do Projeto](https://img.shields.io/badge/Status-Conclu%C3%ADdo-brightgreen)
![Nota](https://img.shields.io/badge/Nota-10%2F10-gold)

Este repositório contém a documentação e o protótipo de alta fidelidade do **SGPD**, um sistema focado em otimizar a abertura e o acompanhamento de demandas internas. O projeto foi desenvolvido como parte do curso de **Análise e Desenvolvimento de Sistemas (Unipê)**, unindo conceitos de Engenharia de Software e Design de Experiência do Usuário (UX).

## 📌 Sobre o Projeto

O SGPD nasceu da necessidade de centralizar processos administrativos. O foco foi criar uma interface que reduza a carga cognitiva do utilizador e garanta a integridade dos dados através de fluxos claros e previsíveis.

### Principais Funcionalidades Proteotipadas:
- **Autenticação e Perfis:** Diferenciação entre Solicitante e Analista.
- **Abertura de Demandas:** Fluxo intuitivo de preenchimento de formulários.
- **Fila de Trabalho:** Visão do analista para triagem e decisão.
- **Dashboard de Acompanhamento:** Status em tempo real dos processos.

---

## 🧠 Engenharia e UX: O Diferencial

O grande diferencial deste projeto foi a aplicação de **Validação Iterativa** e as **Heurísticas de Nielsen**. 

### Ciclo de Validação (Teste de Guerrilha)
Realizei testes reais com utilizadores para identificar gargalos de navegação. O principal achado foi a falta de visibilidade do status do sistema após o envio de formulários.

### Melhorias Implementadas (Refatoração de Interface):
1. **Modal de Sucesso (Heurística #1):** Implementação de feedback visual imediato após o envio, eliminando a incerteza do utilizador.
2. **Redirecionamento Estratégico:** Finalização do ciclo de tarefa com retorno automático à lista de processos, validando a persistência dos dados.

> *"Validar cedo para reduzir o custo de retrabalho e mitigar riscos técnicos antes da fase de codificação."*
